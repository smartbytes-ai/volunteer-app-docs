API Documentation

Features

  • People can create organizations
  • Each organization will be given a join code
  • Students use the join code to join a specific organization
  • The owner of the organization will be able to add hours to a specific student
  • Hours are displayed for each organization (everyone's hours are public)
  • Hours are also displayed on a student's profile with a breakdown of where they got their hours from

IMPORTANT: Once a user logs in, store the token since most of the API requests require the token for authentication. Once the token expires, force the user to login again.

User Interaction Timeline

  1. User is prompted to Sign up or Login
  2. Store the login token
  3. Show the homepage where it will show the amount of hours the user has and the breakdown of the origin of the hours
  4. "+" button on the top to allow user to join a new organization
  5. Code given by the administrator is entered by the student. Once the user joins, they can see the owner and all the students with their respective hours
  6. Owner UI will look slightly different as they will be able to add hours to each student
  7. When hours are added, the profile hours and the hours on the organization are updated

GET /

Returns a simple greeting message.

Response:

"Hello, World!"

POST /api/login

Logs in a user and returns a JWT token.

Input: JSON object with the following fields:

{
    "email": "string",
    "password": "string"
}

Response: JSON object with the following fields:

{
    "token": "string",
    "expires": "string"
}

POST /api/signup

Signs up a new user.

Input: JSON object with the following fields:

{
    "email": "string",
    "password": "string",
    "name": "string"
}

Response: JSON object with a message indicating success or failure.

POST /api/createOrganization

Creates a new organization.

Input: JSON object with the following fields:

{
    "name": "string",
    "token": "string",
    "location": "string",
    "phone": "string"
}

Response: JSON object with a message indicating success or failure.

POST /api/joinOrganization

Joins an existing organization using a code.

Input: JSON object with the following fields:

{
    "code": "string",
    "token": "string"
}

Response: JSON object with a message indicating success or failure.

POST /api/addHours

Adds volunteer hours for a user in an organization.

Input: JSON object with the following fields:

{
    "token": "string",
    "hours": "number",
    "user_id": "string",
    "org_id": "string"
}

Response: JSON object with a message indicating success or failure.

GET /api/getUserProfile

Gets the profile of a user by their user ID.

Input: JSON object with the following fields:

{
    "user_id": "string"
}

Response: JSON object containing user profile data (excluding email and password).

GET /api/getOrganizationData

Gets the data of an organization by its organization ID.

Input: JSON object with the following fields:

{
    "org_id": "string"
}

Response: JSON object containing organization data.
